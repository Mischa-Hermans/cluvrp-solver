# CluVRP вЂ“ Decomposition-Based Matheuristic Solver

This repository contains an implementation of a matheuristic framework for the **Clustered Vehicle Routing Problem (CluVRP)**, covering both the **soft** and **hard** variants.

The approach combines:
- initial solution generation using capacitated clustering
- exact TSP route evaluation per vehicle using Gurobi,
- metaheuristics such as Simulated Annealing (SA), Iterated Local Search (ILS), and Hybrid Genetic Search (HGS) to search the cluster-vehicle assignment space.

---

## Overview

The CluVRP extends the capacitated vehicle routing problem by grouping customers into clusters. A feasible solution assigns each cluster to a vehicle and determines a route for each vehicle.

The implemented approach follows a decomposition strategy:

- **Search space:** cluster-to-vehicle assignments  
- **Evaluation:** exact TSP per vehicle  
- **Improvement:** cluster-level neighborhoods  
- **Metaheuristics:** SA, ILS, HGS  

This allows the algorithm to focus solely on assignment decisions while solving routing accurately.


Each folder contains `.gvrp` files such as `A.gvrp`, `B.gvrp`, and so on.

```text

```

---

## Soft and Hard CluVRP

The same search framework can be used for both variants of the CluVRP. The difference lies in the **route evaluation step**:

- **Soft CluVRP:** standard TSP route evaluation  
- **Hard CluVRP:** TSP route evaluation with the additional constraint that customers in a cluster are visited consecutively 

To switch between the soft and hard variant, update the routing configuration in:

```text
configs/routing.py
```

## Running the Code

### Run a Single Instance

```bash
py -m scripts.run_single
```

This will:
- load a selected instance,
- construct an initial solution,
- improve it using the Simulated Annealing metaheuristic search over assignments,
- print the final solution cost.

To change the instance, seed, time limit, or method, edit:

```text
scripts/run_single.py
```

There are also other metaheuristics scripts:

```bash
py -m scripts.run_single_hgs
py -m scripts.run_single_ils
```

---

### Run the Full Benchmark

```bash
py -m scripts.run_benchmark
```

This will:
- run all selected instances with Simulated Annealing search over assignments,
- execute multiple seeds per instance,
- apply the configured method,
- save results in the `results/` folder.

Other metaheuristics:

```bash
py -m scripts.run_benchmark_hgs
py -m scripts.run_benchmark_ils
```

To change instances, seeds, or time limits, edit the scripts and/or:

```text
configs/
```
For example ```configs/benchmark.py``` looks like this:

```python
"""Settings for benchmark runs and time checkpoints."""

CHECKPOINT_SECONDS = [1, 10, 60, 300]
BENCHMARK_TIME_LIMIT_SECONDS = 300
BASE_SEED = 42
```
while ```configs/sa.py``` lets you change the settings for Simulated Annealing

```python
"""Settings for simulated annealing."""

ALPHA_BALANCE = 0.15
CONSTRUCTION_ITERATIONS = 44

SA_INITIAL_TEMP = 483.8485
SA_COOLING_RATE = 0.9211
SA_ITERATIONS_PER_TEMP = 84
SA_MIN_TEMP = 1e-3
SA_MAX_NEIGHBOR_ATTEMPTS = 250

NEIGHBORHOOD_WEIGHTS = {
    "relocate_best": 1.0,
    "remove_reinsert_three": 1.0,
}
```

## Method Description

1. **Construction**  
   Build an initial assignment using a randomized greedy supercluster method.

2. **Neighborhood Search**  
   Improve assignments using operators such as relocate, swap, and removeвЂ“reinsert.

3. **Exact Route Evaluation**  
   Solve a TSP for each vehicle (soft or hard variant).

4. **Metaheuristic Control**  
   Guide the search using SA, ILS, or HGS.
